Pixel-cap mode v2 — dynamic wrappers (no delta compensation)

Same setup: factor=2, totalLines=50, viewport=10 RH, spacer=30 RH, translateY = −scrollTop.

What changed from v1: wrappers no longer use delta-compensated static positions. Instead:
  • wrapper top = calc(lineIdx*RH + var(--ty)) — wrapper translates with scroll, same as flat rows
  • wrapper height = (subtree − 1) × RH — uncompressed
  • Interior rows: top = (K − parentLineIdx) × RH — static, simple
  • Close row: top = wrapperHeight
The wrapper's pre-activation position now matches the line's true doc position, so the "B floats too high" issue should be gone.
