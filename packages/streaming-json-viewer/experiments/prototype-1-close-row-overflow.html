JSON-mirroring sticky push-up — does the close-row overflow trick work?

Each group is position:absolute; its open row is position:sticky; its close row is also inside the group at top: H_w (= group height = (subtreeLines − 1) × ROW_HEIGHT), paint via default overflow:visible. Group height ends at the close row's top — so the sticky open's containing block ends at the close, and CSS push-up fires when the close enters the pin slot.

What to verify by scrolling

  1. At top: only the root open { sticks (depth 0).
  2. Scroll into group A: "a": [ pins at depth 1, just below root open. Group A primitives scroll past beneath both stickies.
  3. As you reach group A's close ]: it should push "a": [ up, sliding it under the root {. The close itself slides up into the depth-1 slot momentarily, then continues scrolling.
  4. Mid-section (primR1–primR3): only the root { is pinned.
  5. Group B's open "b": [ pins at depth 1.
  6. Group B's close pushes its open up, same as A.
  7. Finally, root close } pushes the root { up at the very end.

If push-up fires at the wrong moment (e.g. close visually overtakes the sticky before push-up starts, or sticky stays pinned while close is already past it), the trick is broken.
